### 1.實體層 (Physical Layer) - L1
- **關鍵字：** **位元 (Bits)**、訊號、介質。
- **職責：** 負責傳送 0 和 1 的電子訊號或光訊號。不在此層談論「資料意義」，只管訊號有沒有送到。
- **常見設備/協議：** 網路線 (RJ45)、光纖、WiFi 電波、集線器 (Hub)。
- **表現 (故障特徵)：**
    - 網卡燈不亮。
    - 線路接觸不良導致頻繁斷線。
    - **生活比喻：** 這裡就像是「馬路」和「卡車」。如果路斷了，貨物（資料）絕對送不過去。
- P.S. 集線器(HUB)已經不存在，其構造像是電路並聯，但不完全是假設一個A設備占用頻寬(1650W)那其他設備想用網路(電)就需要等他用完。

### 2.資料連結層 (Data Link Layer) - L2
- **關鍵字：** **訊框 (Frames)**、MAC Address、交換 (Switching)。
- **職責：** 確保資料能在「同一個區域網路」內正確傳遞。利用 MAC 位址來識別硬體。
- **常見設備/協議：** 交換器 (Switch)、網卡 (NIC)、ARP 協議。
- **表現 (故障特徵)：**
    - 電腦顯示「無法辨識的網路」。
    - MAC 位址衝突（極少見但會導致網路癱瘓）。
    - **生活比喻：** 這是社區內的「門牌號碼」。快遞員（Switch）知道怎麼把包裹送到社區內的某一戶，但他不知道怎麼送到另一個城市。

### 3. 網路層 (Network Layer) - L3
- **關鍵字：** **封包 (Packets)**、IP Address、路由 (Routing)。
- **職責：** 負責跨網路的尋址與路徑選擇。決定資料該走哪條路才能到達世界另一端的伺服器。
- **常見設備/協議：** 路由器 (Router)、IP、ICMP (Ping)。
- **表現 (故障特徵)：**
    - 能連上隔壁電腦，但連不上 Google (外網不通)。
    - `Ping` 顯示 "Destination Host Unreachable"。
    - **生活比喻：** 這是「全球郵政系統」。你寫上地址 (IP)，郵局 (Router) 幫你決定包裹要搭飛機還是搭船轉運。
- P.S. **ICMP補充** : 在 OpenWrt 的防火牆（通常是基於 `iptables` 或 `nftables`）介面中，你會看到這三個選項並列，這是因為從 **「流量過濾」** 的角度來看，它們是同樣層級的判斷基準：
1. **管理邏輯：** 對於防火牆來說，它檢查一個封包時，會看「協定類型」。這就像進城檢查哨，警衛會問：「你是 TCP 貨車、UDP 摩托車，還是 ICMP 公務車？」雖然它們層級略有不同，但都是警衛要攔下來檢查的對象。
2. **安全性考量：** * **TCP/UDP** 決定了哪些「服務」可以被存取。 
    - **ICMP** 決定了你的設備是否能被「看見」。很多管理員會選擇「Drop ICMP」，這樣別人在外網 `ping` 你的 IP 時會沒反應，增加一點隱密性（這就是為什麼有些網站你上得去，但你 ping 不到它）。

### 4. 傳輸層 (Transport Layer) - L4
- **關鍵字：** **區段 (Segments)**、Port、TCP/UDP。
- **職責：** 確保端對端的連線品質。區分這筆資料是要給瀏覽器的 (Port 80/443) 還是給遊戲的。
- **常見設備/協議：** TCP、UDP、防火牆 (Firewall)。
- **表現 (故障特徵)：**
    - 伺服器 Ping 得到，但網頁打不開 (Port 被擋)。
    - 看影片一直緩衝 (封包遺失，TCP 重傳機制運作中)。
    - **生活比喻：** 這是公司的「收發室」。包裹到了大樓 (IP)，收發室看上面的標籤 (Port)，決定送到會計部還是業務部

### L5, 6, 7 在現代網路 (TCP/IP 模型) 中，通常被合併視為「應用層」的範疇

### 5. 會話層 (Session Layer) - L5
- **關鍵字：** 會話 (Session)、對話控制。
- **職責：** 建立、管理和終止應用程式之間的連線。確保即使斷線重連，也能知道你是誰。
- **表現 (故障特徵)：**
    - 登入網銀，放著不動 10 分鐘後被踢出 (Session Timeout)。
    - 下載大檔案斷線後，可以按「續傳」而不是重頭下載 (斷點續傳)。
    - **生活比喻：** 電話接通後的「通話狀態」。如果你掛電話再打過去，對方知道你是剛才那個人，還是把你當陌生人？

### 6. 表現層 (Presentation Layer) - L6
- **關鍵字：** 格式轉換、編碼、加密/解密。
- **職責：** 當翻譯官。確保一個系統傳送的資料，另一個系統能讀懂。處理圖片格式、文字編碼、資料壓縮。
- **表現 (故障特徵)：**
    - 打開網頁看到一堆亂碼 (編碼錯誤，如 UTF-8 vs Big5)。
    - HTTPS 憑證錯誤，瀏覽器顯示「不安全」(加密失敗)。
    - 圖片無法顯示 (格式不支援)。
    - **生活比喻：** 翻譯與包裝。對方寄來的是法文信（加密內容），這層負責把它翻譯成中文（解密）給你讀。

###  7. 應用層 (Application Layer) - L7
- **關鍵字：** 數據 (Data)、介面。
- **職責：** 使用者直接互動的軟體介面。
- **常見設備/協議：** HTTP/HTTPS (網頁), FTP (檔案), SMTP (郵件), DNS (網址解析)。
- **表現 (故障特徵)：**
    - 瀏覽器顯示 404 Not Found (找不到檔案)。
    - 瀏覽器顯示 500 Internal Server Error (程式寫壞了)。
    - **生活比喻：** 這是你手上的「信紙內容」。你寫的內容是什麼，只有這裡在乎。
